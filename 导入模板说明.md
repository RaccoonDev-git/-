# 📋 导入模板说明文档

## 概述
本系统支持多种数据的批量导入功能，每种数据类型都有对应的导入模板。本文档详细说明所有可用的导入模板及其使用方法。

---

## 🎯 可用导入模板列表

### 1️⃣ 学生信息导入模板

#### 📍 **下载方式**
- **API端点**: `GET /api/admin/students/import/template`
- **前端操作**: 教师页面 → 学生管理 → 批量导入 → 选择"学生信息" → 下载模板
- **文件格式**: Excel (.xlsx)
- **文件名**: `student_import_template.xlsx`

#### 📊 **模板结构**
| 列名 | 是否必填 | 数据类型 | 说明 | 示例 |
|------|---------|----------|------|------|
| 学号 | ✅ 必填 | 字符串 | 学生唯一标识 | 2023001 |
| 姓名 | ✅ 必填 | 字符串 | 学生姓名 | 张三 |
| 年级 | ✅ 必填 | 数字 | 入学年份 | 2023 |
| 班级 | ✅ 必填 | 字符串 | 班级名称 | 23软工A1 |
| 专业 | ❌ 可选 | 字符串 | 专业名称 | 软件工程 |
| 手机号 | ❌ 可选 | 字符串 | 11位手机号 | 13800138001 |
| 邮箱 | ❌ 可选 | 字符串 | 电子邮箱 | student@example.com |
| 备注 | ❌ 可选 | 字符串 | 其他说明 | 示例学生 |

#### 📝 **示例数据**
```excel
学号(必填)   姓名(必填)   年级(必填)   班级(必填)   专业         手机号         邮箱                     备注
2023001     张三         2023        23软工A1    软件工程     13800138001   zhangsan@example.com    示例学生1
2023002     李四         2023        23软工A1    软件工程     13800138002   lisi@example.com        示例学生2
2023003     王五         2023        23软工B1    软件工程     13800138003   wangwu@example.com      示例学生3
```

#### ⚠️ **注意事项**
1. **学号唯一性**: 学号必须在系统中唯一，重复学号会导致导入失败
2. **数据格式**: 
   - 年级必须是4位数字（如2023、2024）
   - 手机号必须是11位数字
   - 邮箱必须符合邮箱格式
3. **班级命名**: 建议使用统一的班级命名规范
4. **字符编码**: 支持中文，请使用UTF-8编码
5. **模板行**: 第1行为说明，第2行为表头，从第3行开始填写数据

### 2️⃣ 综合成绩导入模板（通用版）

#### 📍 **下载方式**
- **API端点**: `GET /api/comprehensive-grades/template`
- **前端操作**: 教师页面 → 学生管理 → 批量导入 → 选择"综合成绩" → 不选课程类型 → 下载模板
- **文件格式**: CSV (.csv)
- **文件名**: `comprehensive_grade_import_template.csv`

#### 📊 **模板结构**
| 列名 | 是否必填 | 数据类型 | 说明 | 示例 |
|------|---------|----------|------|------|
| 学号 | ✅ 必填 | 字符串 | 学生学号 | 2023001 |
| 课程ID | ✅ 必填 | 数字 | 课程唯一标识 | 1 |
| 学期 | ✅ 必填 | 字符串 | 学期标识 | 2024-1 |
| 平时分 | ❌ 可选 | 数字 | 0-100分 | 85.5 |
| 期中成绩 | ❌ 可选 | 数字 | 0-100分 | 88.0 |
| 期末成绩 | ❌ 可选 | 数字 | 0-100分 | 90.0 |
| 综合成绩 | ✅ 必填 | 数字 | 0-100分 | 87.8 |

#### 📝 **示例数据**
```csv
学号,课程ID,学期,平时分,期中成绩,期末成绩,综合成绩
2023001,1,2024-1,85.5,88.0,90.0,87.8
2023002,1,2024-1,78.0,82.5,85.0,81.8
2023003,1,2024-1,92.0,95.0,93.5,93.5
```

#### ⚠️ **注意事项**
1. **分数范围**: 所有成绩必须在0-100之间
2. **学期格式**: 格式为 "年份-学期序号"，如"2024-1"（第一学期）、"2024-2"（第二学期）
3. **课程ID**: 必须是系统中已存在的课程ID
4. **学号匹配**: 学号必须是系统中已存在的学生学号
5. **综合成绩计算**: 如果提供了平时分、期中、期末成绩，综合成绩应该是加权平均值

### 3️⃣ 课程专用成绩导入模板

#### 📍 **下载方式**
- **API端点**: `GET /api/templates/preset/{courseType}`
- **前端操作**: 教师页面 → 学生管理 → 批量导入 → 选择"综合成绩" → 选择课程类型 → 下载模板

- **文件名**: `{课程类型}_template.csv`

#### 🎓 **支持的课程类型**
| 课程类型 | 说明 | 特殊字段 |

| 数据结构 | 包含作业成绩 | 作业1, 作业2, 作业3, 作业4 |
| 软件工程 | 包含项目成绩 | 项目1, 项目2, 团队协作 |

#### 📊 **模板结构（以编译原理为例）**
| 列名 | 是否必填 | 数据类型 | 说明 |
|------|---------|----------|------|
| 学号 | ✅ 必填 | 字符串 | 学生学号 |
| 课程ID | ✅ 必填 | 数字 | 课程ID |
| 学期 | ✅ 必填 | 字符串 | 学期 |
| 出勤分 | ❌ 可选 | 数字 | 出勤成绩 |
| 作业分 | ❌ 可选 | 数字 | 作业成绩 |
| 实验1 | ❌ 可选 | 数字 | 第一个实验 |
| 实验2 | ❌ 可选 | 数字 | 第二个实验 |
| 实验3 | ❌ 可选 | 数字 | 第三个实验 |
| 期中成绩 | ❌ 可选 | 数字 | 期中考试 |
| 期末成绩 | ❌ 可选 | 数字 | 期末考试 |
| 综合成绩 | ✅ 必填 | 数字 | 最终成绩 |

### 4️⃣ 动态课程模板

#### 📍 **下载方式**
- **API端点**: `GET /api/templates/course/{courseId}`
- **功能**: 根据特定课程ID生成专属模板
- **文件格式**: CSV (.csv)
- **文件名**: `course_{courseId}_template.csv`

#### 💡 **特点**
- 自动根据课程配置生成模板字段
- 包含该课程的所有评分项
- 动态适应不同课程的评分标准

---

### 5️⃣ 通用数据导入模板

#### 📍 **下载方式**
- **API端点**: `GET /api/templates/generic`
- **功能**: 包含所有可能的评分项的通用模板
- **文件格式**: CSV (.csv)
- **文件名**: `generic_template.csv`

#### 📊 **模板结构**
包含以下所有字段：
- 基础信息: 学号、课程ID、学期
- 平时成绩: 出勤分、作业分、课堂参与
- 实验成绩: 实验1-5
- 考试成绩: 期中、期末
- 项目成绩: 项目1-3
- 最终成绩: 综合成绩

## 🔧 导入流程

### 步骤1: 下载模板
1. 登录系统（管理员或教师账号）
2. 进入教师页面 → 学生管理
3. 点击"批量导入"按钮
4. 选择导入类型（学生信息 或 综合成绩）
5. 如果是综合成绩，可选择课程类型
6. 点击"下载模板"按钮

### 步骤2: 填写数据
1. 打开下载的模板文件
2. 按照模板格式填写数据
3. 确保必填字段都已填写
4. 检查数据格式是否正确
5. 保存文件

### 步骤3: 上传导入
1. 在导入模态框中点击"选择文件"或拖拽文件
2. 选择填写好的数据文件
3. 系统会显示文件名和预览信息
4. 确认无误后点击"开始导入"
5. 等待导入完成，查看导入结果

## ⚙️ API端点总览

### 学生相关
```
GET  /api/admin/students/import/template          - 下载学生导入模板
POST /api/admin/students/import                   - 批量导入学生
GET  /api/admin/students/export                   - 导出所有学生
POST /api/admin/students/export/selected          - 导出选中学生
GET  /api/admin/students/{studentId}/export       - 导出单个学生
```

### 成绩相关
```
GET  /api/comprehensive-grades/template           - 下载综合成绩模板（通用）
POST /api/comprehensive-grades/import             - 批量导入综合成绩
GET  /api/comprehensive-grades/course/{courseId}/export - 导出课程成绩
GET  /api/comprehensive-grades/student/{studentId}/export - 导出学生成绩
```

### 动态模板
```
GET  /api/templates/course/{courseId}             - 生成课程专用模板
GET  /api/templates/preset/{courseType}           - 生成预设类型模板
GET  /api/templates/generic                       - 生成通用模板
GET  /api/templates/course-types                  - 获取支持的课程类型列表
```

## ❌ 常见错误及解决方案

### 1. 导入失败：学号重复
**原因**: 导入的学号在系统中已存在  
**解决**: 检查数据文件，移除或修改重复的学号

### 2. 导入失败：必填字段为空
**原因**: 必填字段（学号、姓名、年级、班级等）未填写  
**解决**: 补充所有必填字段的数据

### 3. 导入失败：数据格式错误
**原因**: 数据格式不符合要求（如年级不是数字、手机号格式错误）  
**解决**: 检查并修正数据格式

### 4. 下载模板失败：权限不足
**原因**: 未登录或Token过期  
**解决**: 重新登录系统

### 5. 导入失败：课程ID不存在
**原因**: 填写的课程ID在系统中不存在  
**解决**: 使用系统中已存在的课程ID

### 6. 导入失败：文件格式不支持
**原因**: 上传的文件格式不在支持列表中  
**解决**: 使用支持的文件格式（.xlsx, .xls, .csv, .json）

## 💡 最佳实践

### 数据准备
1. **先导入学生信息，再导入成绩数据**
2. **使用统一的命名规范**（班级名、专业名等）
3. **小批量测试**：先导入少量数据测试，确认无误后再批量导入
4. **数据备份**：导入前备份现有数据

### 数据质量
1. **去除空行**：删除Excel中的空白行
2. **统一格式**：确保同类数据格式一致
3. **数据验证**：导入前在Excel中验证数据的合理性
4. **特殊字符**：避免使用特殊字符和emoji

### 安全建议
1. **权限控制**：只授予必要人员导入权限
2. **审计记录**：保留导入操作的日志记录
3. **数据隐私**：处理敏感数据时注意保护隐私
4. **定期清理**：定期清理测试数据
