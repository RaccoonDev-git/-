# 📋 自动数据格式转换功能说明

## 🎯 功能概述

学生学情分析系统现在支持**自动数据格式转换**功能！在导入学生和成绩数据时，系统会自动识别并修正各种不规范的数据格式，大大提高了数据录入的容错性和便捷性。

### ✨ 核心特性

- ✅ **自动字段名标准化** - 识别各种字段名变体
- ✅ **智能数据清洗** - 自动移除无效字符和空白
- ✅ **多格式分数支持** - 支持百分比、分数、小数等多种格式
- ✅ **手机号自动识别** - 提取有效的 11 位手机号
- ✅ **学号格式统一** - 自动提取和标准化学号
- ✅ **实时转换反馈** - 显示有多少数据被自动修正

## 📊 支持的数据格式转换

### 1. 字段名（列名）自动识别

系统会自动识别以下字段名的各种变体：

#### 姓名字段

```
支持的格式：
✅ 姓名, 名字, 学生姓名, name
  ↓ 自动转换为
✅ 姓名
```

#### 学号字段

```
支持的格式：
✅ 学号, 学生学号, studentnumber, studentid, 学生编号, 编号
  ↓ 自动转换为
✅ 学号
```

#### 年级字段

```
支持的格式：
✅ 年级, 入学年份, grade
  ↓ 自动转换为
✅ 年级
```

#### 班级字段

```
支持的格式：
✅ 班级, class, 班
  ↓ 自动转换为
✅ 班级
```

#### 专业字段

```
支持的格式：
✅ 专业, major, 所属专业
  ↓ 自动转换为
✅ 专业
```

#### 手机号字段

```
支持的格式：
✅ 手机号, 联系方式, 电话, 手机, phone, mobile, 联系电话
  ↓ 自动转换为
✅ 手机号
```

#### 备注字段

```
支持的格式：
✅ 备注, 说明, note, remark
  ↓ 自动转换为
✅ 备注
```

### 2. 姓名格式标准化

```
原始数据              →  标准化后
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
张  三               →  张三
Zhang San            →  ZhangSan
张三123              →  张三
  王五               →  王五
```

**处理规则：**

- ✅ 移除所有空白字符
- ✅ 只保留中文和英文字母
- ✅ 长度必须在 2-50 个字符之间

### 3. 学号格式标准化

```
原始数据              →  标准化后
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
No.20191001          →  20191001
学号：2019-1001      →  20191001
STU20191001          →  20191001
2019.10.01           →  20191001
```

**处理规则：**

- ✅ 提取所有数字字符
- ✅ 长度必须在 4-20 位之间
- ✅ 自动移除前缀、分隔符

### 4. 年级格式标准化

```
原始数据              →  标准化后
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
19级                 →  2019
2019级               →  2019
19届                 →  2019
95级                 →  1995
```

**处理规则：**

- ✅ 自动提取 4 位年份
- ✅ 两位数年份自动补全（19→2019, 95→1995）
- ✅ 移除"级"、"届"等后缀

### 5. 班级格式标准化

```
原始数据              →  标准化后
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
软工1班              →  软工1
19软工A1班           →  19软工A1
计算机 1 班          →  计算机1
```

**处理规则：**

- ✅ 移除多余空格
- ✅ 自动移除"班"后缀
- ✅ 统一格式

### 6. 手机号格式标准化

```
原始数据                    →  标准化后
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
138-0013-8000              →  13800138000
138 0013 8000              →  13800138000
+86 138 0013 8000          →  13800138000
(+86)13800138000           →  13800138000
0086-138-0013-8000         →  13800138000
```

**处理规则：**

- ✅ 移除所有非数字字符
- ✅ 自动移除国家代码（+86, 0086）
- ✅ 验证 11 位且以 1 开头
- ✅ 支持各种分隔符格式

### 7. 分数格式标准化（重点！）

#### 格式 1：纯数字

```
原始数据    →  标准化后
━━━━━━━━━━━━━━━━━━━━━
85          →  85
85.5        →  85.5
90.25       →  90.25
```

#### 格式 2：百分比

```
原始数据    →  标准化后
━━━━━━━━━━━━━━━━━━━━━
85%         →  85
90.5%       →  90.5
100%        →  100
```

#### 格式 3：带单位

```
原始数据    →  标准化后
━━━━━━━━━━━━━━━━━━━━━
85分        →  85
90.5分      →  90.5
85 points   →  85
90 point    →  90
```

#### 格式 4：分数形式（重点！）

```
原始数据       →  计算过程         →  标准化后
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
85/100        →  85÷100×100      →  85
42.5/50       →  42.5÷50×100     →  85
34/40         →  34÷40×100       →  85.0
17/20         →  17÷20×100       →  85.0
```

**自动百分制转换：**
系统会自动将非百分制的分数转换为百分制！

#### 格式 5：特殊值处理

```
原始数据                →  标准化后
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
-                       →  null（跳过）
null                    →  null（跳过）
缺考                    →  null（跳过）
absent                  →  null（跳过）
空单元格                →  null（跳过）
```

**处理规则：**

- ✅ 支持 0-150 分范围
- ✅ 自动识别百分比格式
- ✅ 自动转换分数形式为百分制
- ✅ 保留一位小数（四舍五入）
- ✅ 处理缺考等特殊情况

## 🔧 使用示例

### 示例 1：混乱格式的学生数据

**原始 Excel/CSV 数据：**

```csv
name,studentnumber,grade,class,phone,数学,英语
张  三,No.20191001,19级,软工1班,138-0013-8000,85分,90%
李四,学号2019-1002,2019,19软工A2班,+86 139 0013 9000,42.5/50,88
```

**系统自动转换后：**

```csv
姓名,学号,年级,班级,手机号,数学,英语
张三,20191001,2019,软工1,13800138000,85,90
李四,20191002,2019,19软工A2,13900139000,85.0,88
```

**转换说明：**

1. ✅ `name` → `姓名`
2. ✅ `studentnumber` → `学号`
3. ✅ `No.20191001` → `20191001`（移除前缀）
4. ✅ `19级` → `2019`（补全年份）
5. ✅ `软工1班` → `软工1`（移除"班"）
6. ✅ `138-0013-8000` → `13800138000`（移除分隔符）
7. ✅ `85分` → `85`（移除单位）
8. ✅ `90%` → `90`（百分比转换）
9. ✅ `42.5/50` → `85.0`（分数转百分制）

## 📝 数据验证规则

### 必填字段验证

| 字段   | 要求             | 错误处理                  |
| ------ | ---------------- | ------------------------- |
| 姓名   | 2-50 个字符      | 跳过该条记录              |
| 学号   | 4-20 位数字      | 跳过该条记录              |
| 手机号 | 11 位且以 1 开头 | 设置为 null，但不影响导入 |
| 分数   | 0-150 范围       | 超出范围的分数会被跳过    |

### 自动修正策略

```
数据验证流程：
1. 原始数据读取
   ↓
2. 字段名标准化
   ↓
3. 字段值清洗
   ↓
4. 格式转换
   ↓
5. 数据验证
   ↓
6. 验证通过 → 导入
   验证失败 → 跳过并记录错误
```

## ⚠️ 注意事项

### 1. 数据质量建议

虽然系统支持自动转换，但建议：

- ✅ 尽量使用标准格式的数据
- ✅ 检查转换后的数据是否符合预期
- ✅ 重要数据导入前先用小样本测试

### 2. 无法自动修正的情况

以下情况系统无法自动修正，会跳过该记录：

- ❌ 姓名完全是特殊字符或数字
- ❌ 学号少于 4 位或超过 20 位
- ❌ 分数超出 0-150 范围
- ❌ 手机号不是 11 位或不以 1 开头

### 3. 分数转换精度

- 分数转换为百分制时保留 1 位小数
- 使用四舍五入规则
- 例如：`17/20 = 85.0`，不是 `85.00000...`

### 4. 性能考虑

- 数据标准化会增加少量处理时间
- 对于大批量数据（1000+条），建议分批导入
- 系统会自动优化重复的转换操作
