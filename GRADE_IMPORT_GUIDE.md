# 📊 学生成绩导入功能使用指南

## 🎯 功能概述

学生学情分析系统现在支持**一次性导入学生基本信息和成绩数据**，系统会自动：

- ✅ 创建学生账号
- ✅ 导入学生基本信息
- ✅ 自动创建课程（如果不存在）
- ✅ 导入成绩记录
- ✅ 支持 CSV、Excel、JSON 三种格式

## 📋 支持的数据格式

### 方式 1：课程名称作为列名（推荐）

**CSV 格式示例**：

```csv
姓名,学号,年级,班级,专业,手机号,数学,英语,计算机基础,数据结构,备注
张三,20191001,2019,19软工A1,软件工程,13800138000,85,90,88,92,优秀学生
李四,20191002,2019,19软工A1,软件工程,13900139000,78,82,85,80,
```

**Excel 格式示例**：

|------|------|------|------|------|--------|------|------|-----------|---------|------|
| 张三 | 20191001 | 2019 | 19 软工 A1 | 软件工程 | 13800138000 | 85 | 90 | 88 | 92 | 优秀学生 |
| 李四 | 20191002 | 2019 | 19 软工 A1 | 软件工程 | 13900139000 | 78 | 82 | 85 | 80 | |

**JSON 格式示例**：

```json

    "年级": "2019",
    "班级": "19软工A1",
```

## 📊 字段说明

### 必填字段

| 字段名 | 说明     | 示例     |
| ------ | -------- | -------- |
| 姓名   | 学生姓名 | 张三     |
| 学号   | 唯一学号 | 20191001 |

### 可选基本信息字段

| 字段名 | 说明     | 示例        |
| ------ | -------- | ----------- |
| 年级   | 入学年份 | 2019        |
| 班级   | 班级名称 | 19 软工 A1  |
| 专业   | 专业名称 | 软件工程    |
| 手机号 | 联系方式 | 13800138000 |
| 备注   | 其他信息 | 优秀学生    |

### 成绩字段（动态）

- **格式**：课程名称作为列名，值为成绩分数
- **示例**：数学=85，英语=90，计算机基础=88
- **支持数量**：无限制（支持任意多门课程）
- **分数范围**：0-150 分（默认满分 100 分）
- **自动功能**：
  - 自动创建不存在的课程
  - 自动计算百分比和等级（A/B/C/D/F）
  - 已存在的学生会更新成绩

## 🚀 使用步骤

## 💡 智能功能

### 1. 自动创建课程

如果导入的成绩中包含系统中不存在的课程，系统会自动创建：

- **课程名称**：从列名获取
- **课程代码**：自动生成（课程名+时间戳）
- **授课教师**：分配给系统第一个教师
- **课程信息**：
  - 学分：3 学分
  - 学期：当前学期
  - 容量：100 人
  - 状态：激活

### 2. 智能成绩识别

系统会自动识别哪些列是成绩列：

- ✅ 排除基本信息字段（姓名、学号、班级等）
- ✅ 排除"课程 X"、"成绩 X"格式的列名
- ✅ 只识别值为有效分数的列
- ✅ 有效分数范围：0-150 分

### 3. 重复数据处理

- **学生已存在**：
  - 跳过学生创建
  - 更新或添加成绩记录
  - 提示"学号已存在，将更新成绩信息"
- **成绩已存在**：
  - 更新现有成绩
  - 保持历史记录

### 4. 自动等级计算

成绩导入后自动计算：

- **百分比**：(得分/满分) × 100%
- **等级**：
  - A：90-100 分
  - B：80-89 分
  - C：70-79 分
  - D：60-69 分
  - F：0-59 分（不及格）

## ⚠️ 注意事项

### 1. 数据格式

- ✅ CSV 文件必须使用 UTF-8 编码
- ✅ Excel 支持.xlsx 和.xls 格式
- ✅ JSON 必须是有效的 JSON 数组格式
- ✅ 列名区分大小写

### 2. 分数有效性

- ✅ 分数范围：0-150
- ✅ 支持小数（如 85.5）
- ✅ 空值会被跳过
- ❌ 非数字会被忽略

### 3. 课程命名

- ✅ 课程名称长度：2-100 字符
- ✅ 支持中英文和数字
- ✅ 建议使用规范的课程名称
- ❌ 避免使用特殊符号

### 4. 学号唯一性

- ✅ 学号必须唯一
- ✅ 重复学号会跳过创建学生
- ✅ 但会更新成绩信息
- ⚠️ 建议先清理数据

## 🐛 常见问题

### Q1: 导入后没有成绩数据？

**A:** 检查以下几点：

1. 确认文件中有课程列（除基本信息外的列）
2. 确认成绩值是有效数字（0-150）
3. 检查列名是否使用了保留字段名
4. 查看导入结果中的 warnings 信息

### Q2: 课程没有自动创建？

**A:** 可能原因：

1. 系统中没有教师账号（需要先创建教师）
2. 列名与保留字段冲突
3. 成绩值无效

**解决方法**：

```sql
-- 检查是否有教师
SELECT * FROM teachers;

-- 如果没有，先创建一个教师账号
```

### Q3: 成绩计算不正确？

**A:** 检查：

1. 满分设置是否为 100（默认）
2. 分数是否超出范围
3. 是否有小数精度问题

### Q4: 导入很慢？

**A:** 优化建议：

1. 单次导入不超过 500 条记录
2. 减少课程数量（每个学生）
3. 使用 CSV 格式（比 Excel 快）
4. 分批导入大量数据

### Q5: 学生已存在但成绩未更新？

**A:** 检查：

1. 学号是否完全匹配
2. 查看 warnings 中的提示信息
3. 确认用户有导入权限
